## Project 1: Command-Line Tool

**Skills:** Python basics, CLI design, file I/O

### Description

Build a useful command-line tool that you'd actually use or others could use.

- [ ] Git and GitHub
- [ ] Code documentation

- Code is clean and well-organized
- README is clear
- Tests pass
- GitHub repo shows your work

## Project Guidelines

### General Requirements (All Projects)

- [ ] **Clean, professional code**
  - Functions and classes well-organized
  - Follows PEP 8
  - Meaningful variable names
  - Comments where complex
- [ ] **Version control**
  - Git repository with meaningful commits
  - Deployed to GitHub
  - README file
- [ ] **Testing**
  - Unit tests for core logic
  - 80%+ coverage on business logic
  - Tests pass consistently
- [ ] **Documentation**
  - README explains project
  - Installation and usage instructions
  - API documentation (if applicable)
  - Docstrings on functions
- [ ] **Error handling**
  - Graceful failure modes
  - Helpful error messages
  - Logging for debugging
- [ ] **Deployment**
  - Can run project from GitHub
  - For projects 3-5: deployed to internet
  - Production-ready (not just local)

### How to Complete Projects

1. **Choose based on interest** - you'll spend weeks on this
2. **Start with scope** - what are you building?
3. **Design before coding** - sketch API, database schema, etc.
4. **Build incrementally** - small features that work
5. **Test as you go** - don't test everything at the end
6. **Document continuously** - document as you code
7. **Deploy early** - get it online sooner
8. **Iterate** - improve based on feedback
9. **Showcase it** - write blog post, share with community
