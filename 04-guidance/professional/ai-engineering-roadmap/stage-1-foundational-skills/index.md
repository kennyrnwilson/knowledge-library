# Stage 1: Foundational Skills

**Goal:** Master Python, Git, APIs, databases, and software engineering fundamentals

**Estimated Time:** 6-12 months part-time (10-15 hours/week)

**Entry Requirements:**
- Basic computer literacy
- Able to follow technical documentation
- Access to a computer with admin rights
- Willingness to practice consistently

## Overview

This stage focuses on building a strong foundation in software engineering. You'll learn Python programming, version control, databases, and web basics - the skills needed before diving into AI.

**Key Insight:** Don't skip this stage unless you're already a strong software engineer. These fundamentals are what separates weekend hobbyists from professional engineers.

## Learning Path

**[Python Setup & Environment](./python-fundamentals/index.md)**
- Installation (Windows, Mac, Linux)
- Virtual environments
- Modern tooling (Poetry, pyproject.toml)
- VS Code setup for Python
- Package management

**[Core Python Concepts](./python-fundamentals/core-concepts.md)**

### 1.2 Software Engineering Practices (2-3 months)

**Git & Version Control**
- Git basics (init, add, commit, push)
- GitHub workflow
- Branching strategies
- Collaborative coding
- `.gitignore` and project structure

**Testing & Code Quality**
- Unit testing with pytest

- Proper file organization

### 1.3 Data & Databases (2-3 months)

**Data Handling**
- CSV files and tabular data
- Pandas basics (reading, filtering, grouping)
- Data visualization (matplotlib, pandas plotting)
- Common data formats (JSON, parquet)

**SQL Fundamentals**
- Database concepts (tables, keys, relationships)
- SQL queries (SELECT, INSERT, UPDATE, DELETE)

**Python + Databases**
- sqlite3 (built-in SQL)
- SQLAlchemy ORM
- Parameterized queries (prevent SQL injection)
- Connection pooling
- Migration tools

### 1.4 Web Development & APIs (2-3 months)

**HTTP & Web Basics**
- How the web works (client-server)
- HTTP methods (GET, POST, PUT, DELETE)
- Status codes (200, 404, 500, etc.)
- Headers and requests
- JSON format

**APIs in Python**
- requests library
- Making GET/POST requests
- Handling responses and errors
- API authentication
- Rate limiting

**Building APIs**
- FastAPI fundamentals
- Routing and endpoints
- Request parameters and body
- Response handling
- Error handling
- CORS and middleware

**Advanced API Topics**
- Authentication (API keys, JWT)
- File uploads
- Streaming responses
- Background tasks
- Async/await basics

- Deploying a simple application

- Basic monitoring

## Completion Criteria

You've completed Stage 1 when you can:

- ✅ **Write clean, professional Python code**
  - Follows PEP 8
  - Proper naming conventions
  - Good error handling
  - Fully documented

- ✅ **Use Git and GitHub confidently**
  - Commit regularly
  - Write clear commit messages
  - Push to GitHub
  - Collaborate on projects

- ✅ **Work with databases**
  - Write SQL queries
  - Use Python with SQL
  - Design basic database schemas
  - Prevent SQL injection

- ✅ **Build a REST API from scratch**
  - Create endpoints
  - Handle requests/responses
  - Proper error handling
  - Basic authentication

- ✅ **Deploy an application to the internet**
  - App is publicly accessible
  - Can access logs
  - Can make changes and redeploy

- ✅ **Have 5 substantial GitHub projects**
  - Well-documented
  - Tests included
  - Proper structure
  - Deployed and working
